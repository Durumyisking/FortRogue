# FortRogue Fun Direction

작성일: 2026-06-20 KST

전제:
- 지금 구현을 반드시 고수하지 않는다.
- 목표는 "기존 구조를 살리는 방법"이 아니라 "게임이 더 재미있어지는 방향"을 정하는 것이다.
- FortRogue는 멀티 대전 포격 게임이 아니라 싱글플레이 턴제 지형파괴 로그라이크다.

## 1. 결론

FortRogue는 `Shell을 모으는 게임`이 아니라 `캐릭터 고유 규칙으로 Shell을 다르게 쓰는 게임`이 되어야 한다.

추천 방향:

- 전투는 기본적으로 `플레이어 1명 vs 적 다수`가 좋다.
- 일반 적은 플레이어 캐릭터를 그대로 쓰지 않는다.
- 플레이어블 캐릭터는 Binding of Isaac처럼 시작 조건, 고유 규칙, 제약, 보상 성향이 끝까지 따라가야 한다.
- Shell은 교체 가능해야 한다. 다만 캐릭터성은 Shell 자체가 아니라 `Signature Core`, `Core Passive`, `Reward Bias`, `Constraint`에서 나온다.
- Perk은 단순 스탯 보상이 아니라 캐릭터의 고유 규칙과 Shell을 연결하는 빌드 엔진이어야 한다.
- 스테이지는 단순 적 처치 방이 아니라 매번 다른 포격 퍼즐이어야 한다.

한 줄로 말하면:

> 같은 Shell을 먹어도 캐릭터마다 다른 빌드가 되는 포격 로그라이크.

## 2. 캐릭터성이 사라지는 문제

현재 고민:

- 캐릭터성은 스탯과 시작 무기에서 나온다.
- 그런데 런 중 Shell을 바꾸면 시작 무기 기반 캐릭터성이 사라진다.
- Shell 교체를 막으면 ShellShock식 무기 다양성과 로그라이크 보상 재미가 줄어든다.

해결:

캐릭터성은 `무엇을 들고 시작하는가`가 아니라 `모든 선택을 어떤 방식으로 비트는가`에서 나오게 한다.

캐릭터 정체성 구성:

| 구성 요소 | 설명 | 예시 |
| --- | --- | --- |
| Signature Core | 캐릭터마다 고정되는 고유 장치/기본 Shell. 교체되지 않는다. | Miner는 항상 Drill Core를 가진다. |
| Core Passive | 런 내내 적용되는 고유 규칙. | 지형을 팔 때마다 다음 폭발 반경 증가. |
| Constraint | 캐릭터를 불편하게 만드는 제약. | 직접 피해 낮음, 체력 낮음, 이동력 낮음. |
| Reward Bias | 특정 보상/무기 계열이 더 자주 뜨는 성향. | Miner는 Drill, Quake, Terrain 보상이 자주 뜬다. |
| Character Perks | 해당 캐릭터에게만 뜨는 전용 퍽. | Engineer 전용: 생성한 지형 위에 있으면 방어막. |

중요한 결정:

- Shell 교체는 허용한다.
- 캐릭터의 Signature Core는 교체하지 않는다.
- 일반 Shell 슬롯은 도구이고, 캐릭터 Core가 그 도구의 의미를 바꾼다.

예시:

`Split Shell`을 먹었을 때:

- Miner: child 탄이 더 많이 지형을 파서 낙사 빌드가 된다.
- Engineer: child 탄이 작은 발판을 남겨 방어 빌드가 된다.
- Gunslinger: child 수보다 직격/치명타 보정으로 정밀 빌드가 된다.
- Scavenger: Split 업그레이드 보상을 더 자주 보고 빠르게 완성한다.

## 3. 전투 구성

### 기본값은 1 vs 다수

FortRogue는 싱글플레이 로그라이크이므로 1:1 포격만 반복하면 전투가 쉽게 단조로워진다.

1 vs 다수가 좋은 이유:

- 광역탄, 분열탄, 지형 파괴, 낙사, 위치 조작이 모두 의미를 갖는다.
- 플레이어가 매 턴 "누구를 먼저 처리할까"를 고민한다.
- 적 조합이 스테이지 정체성을 만든다.
- 약한 적 여럿은 플레이어 빌드가 강해졌다는 느낌을 잘 보여준다.
- 보스 전까지도 작은 전술 퍼즐을 계속 만들 수 있다.

추천 구성:

| 전투 타입 | 비율 | 역할 |
| --- | ---: | --- |
| 1 vs 2-4 일반 전투 | 60% | 기본 전투. 적 조합과 지형으로 문제를 만든다. |
| 1 vs 다수 웨이브 | 15% | 광역/지형 빌드가 빛나는 방. |
| 1 vs 엘리트 | 10% | 정확도, 생존, 특정 기믹을 시험한다. |
| 1 vs 라이벌 | 5% | 플레이어블 캐릭터의 그림자 버전. 특별 이벤트로 사용한다. |
| 보스전 | 10% | 지형 룰 자체를 바꾸는 전투. |

### 적은 플레이어 캐릭터를 그대로 쓰지 않는다

일반 적을 플레이어 캐릭터 데이터로 재사용하는 것은 추천하지 않는다.

이유:

- 플레이어 캐릭터는 장기 빌드와 선택을 위해 설계된다.
- 적은 한 스테이지에서 빠르게 읽히는 행동 패턴이 더 중요하다.
- 플레이어와 같은 복잡도를 가지면 AI, 밸런스, UI 설명 비용이 커진다.
- 일반 적이 플레이어처럼 모든 선택지를 가지면 전투 의도가 흐려진다.

대신:

- 일반 적은 전용 Enemy Archetype으로 만든다.
- 엘리트/라이벌/보스만 플레이어 캐릭터의 변형을 사용한다.
- 라이벌은 "플레이어블 캐릭터의 완전 복사"가 아니라 그 캐릭터 테마를 가진 간소화 보스다.

## 4. 적 캐릭터 설계

적은 체력과 피해량만 다른 표적이면 안 된다. 각 적은 플레이어에게 다른 질문을 던져야 한다.

| 적 | 역할 | 플레이어에게 던지는 질문 |
| --- | --- | --- |
| Grunt | 기본 포격병 | 안정적으로 제거할까, 다른 적부터 볼까? |
| Mortar | 고각 포격병 | 엄폐 뒤에 있는 적을 어떻게 처리할까? |
| Burrower | 굴착병 | 발밑 지형이 무너지기 전에 이동할까? |
| Builder | 지형 생성 적 | 방벽을 부술까, 우회할까? |
| Scout | 이동형 적 | 매 턴 위치가 바뀌는 적을 어떻게 맞힐까? |
| Sniper | 정밀 저격 적 | 작은 피해를 누적시키기 전에 끊을까? |
| Bomber | 자폭/근접 압박 | 가까이 오기 전에 밀어낼까? |
| Shield Carrier | 보호막 적 | 보호받는 적을 먼저 우회할까, 방패를 깰까? |
| Repair Drone | 회복/수리 적 | 딜러보다 서포터를 먼저 잡을까? |
| Splitter | 다중탄 적 | 넓은 지역 압박을 피하려면 어디로 움직일까? |
| Rival | 플레이어블 캐릭터 테마 적 | 내 빌드와 비슷한 규칙을 상대하면 어떻게 대응할까? |
| Boss | 지형 룰 변경 | 이 전투의 특수 규칙을 어떻게 이용할까? |

적 AI 원칙:

- 너무 정확하면 불쾌하고, 너무 멍청하면 의미가 없다.
- 적은 다음 행동 의도가 보여야 한다.
- 강한 공격은 예고해야 한다.
- 적 턴은 빠르게 진행되어야 한다.
- 다수 적 전투에서는 모든 적이 매 턴 완벽한 공격을 하면 안 된다. 일부는 이동, 장전, 방어, 준비 행동을 섞는다.

UI 원칙:

- 적 머리 위에 다음 행동 아이콘을 표시한다.
- 강한 공격은 착탄 예상 영역을 간단히 보여준다.
- 적 역할은 실루엣과 색으로 구분한다.

## 5. 스테이지 설계

스테이지는 "적을 모두 죽여라"만 반복하면 안 된다. 각 스테이지는 하나의 전술 질문을 가져야 한다.

### 런 구조

데모 기준 7스테이지:

| Stage | 타입 | 목적 |
| --- | --- | --- |
| 1 | Starter Fight | 캐릭터 고유 규칙을 체감하게 한다. |
| 2 | Choice Fight | 보상 선택이 전투 방식을 바꿨는지 확인한다. |
| 3 | Terrain Puzzle | 지형 파괴/생성/이동 판단을 요구한다. |
| 4 | Elite or Event | 빌드가 약한 부분을 시험한다. |
| 5 | Multi-Enemy Fight | 광역/우선순위 판단을 시험한다. |
| 6 | Shop/Event/Hard Fight | 회복, 구매, 위험 보상 선택을 만든다. |
| 7 | Boss | Act의 지형 규칙을 완성형으로 보여준다. |

정식 구조는 Act 단위로 늘린다. 한 Act는 7-9스테이지가 적당하다.

### 스테이지 타입

| 타입 | 설명 | 재미 |
| --- | --- | --- |
| Skirmish | 기본 전투 | 빌드의 평균 성능을 확인한다. |
| Ambush | 적이 가까이 있거나 위협 위치에서 시작 | 즉시 이동/방어/밀쳐내기 판단. |
| High Ground | 적이 높은 곳에 있음 | 고각, 지형 파괴, 이동 선택. |
| Canyon | 깊은 골짜기와 낙사 위험 | 낙사, 발판 생성, 굴착이 중요. |
| Fortified | 적이 방벽 뒤에 있음 | Drill, Air, Builder 대응. |
| Crate Room | 상자/위험물/보상이 맵에 있음 | 적 처치와 보상 획득 사이 선택. |
| Survival | 일정 턴 버티기 | 방어, 지형 생성, 이동 빌드 가치. |
| Hunt | 도망가는 적 처치 | 이동력, 저각탄, 정밀탄 가치. |
| Elite | 강한 단일 적 | 단일 목표 딜과 생존 시험. |
| Boss | 지형 룰 변경 | Act의 최종 시험. |

### 맵 템플릿

완전 랜덤보다 저작된 템플릿 + 변주가 좋다.

필수 템플릿:

- Flat Ridge: 기본 조준 학습.
- Broken Hills: 고각/저각 선택.
- Deep Canyon: 낙사와 Builder 가치.
- Fort Wall: Drill/Air Support 가치.
- Twin Towers: 지형 파괴로 타워를 무너뜨리는 재미.
- Cracked Floor: 과한 폭발이 자신에게도 위험.
- Wind Gap: 바람을 이용하거나 거스르는 판단.

스테이지 변주 요소:

- 바람 강도
- 지형 내구도
- 폭발성 오브젝트
- 상자 위치
- 적 스폰 높이
- 파괴 시 무너지는 약한 지형
- 이동 제한 구간

## 6. 플레이어 캐릭터 제안

데모는 3-4명으로 충분하다. 정식은 8명 이상까지 확장할 수 있다.

### 데모 우선 캐릭터

| 캐릭터 | 판타지 | Signature Core | Core Passive | Constraint | Reward Bias |
| --- | --- | --- | --- | --- | --- |
| Miner | 지형을 파서 이기는 굴착 전문가 | Drill Core | 지형을 많이 파괴할수록 다음 샷 피해/반경 증가 | 직접 폭발 피해 낮음 | Drill, Quake, Terrain, Drop |
| Engineer | 지형을 만들고 버티는 방어형 | Builder Core | 생성한 지형 근처에 있으면 방어막/회복 | 이동력 낮음 | Builder, Shield, Repair, Support |
| Gunslinger | 작은 표적을 정확히 맞히는 정밀 딜러 | Precision Core | 직격 시 치명타/추가 턴 자원 | 폭발 반경 낮음 | Sniper, Laser, Crit, Low Angle |
| Scavenger | 약하게 시작하지만 선택지가 많은 탐험가 | Salvage Core | 보상 선택지/상자/아이템 획득량 증가 | 시작 체력/피해 낮음 | Luck, Shop, Item, Random Upgrade |

### 정식 확장 캐릭터

| 캐릭터 | 판타지 | Core Passive | Constraint |
| --- | --- | --- | --- |
| Trickshot | 반사와 분열을 계산하는 곡예형 | 벽/지형 반사 후 명중 시 보너스 | 첫 직격 피해 낮음 |
| Stormcaller | 바람을 읽고 조작하는 캐릭터 | 바람 방향으로 쏘면 강화, 일부 아이템으로 풍향 조작 | 무풍 상태에서 약함 |
| Berserker | 위험할수록 강해지는 고위험 캐릭터 | 체력이 낮을수록 폭발 반경/피해 증가 | 회복 효율 낮음 |
| Artillerist | 위에서 떨어뜨리는 고각 포격 전문가 | 고각 발사/상단 낙하 공격 강화 | 근거리 대응 약함 |
| Alchemist | 탄을 상태 이상/장판으로 바꾸는 캐릭터 | 산성, 화염, 연막 같은 잔류 효과 강화 | 순수 피해 낮음 |
| Gambler | 랜덤을 이용하는 운빨 고점 캐릭터 | 보상이 흔들리지만 고희귀 보상 확률 증가 | 저점이 낮음 |

캐릭터 제작 규칙:

- 좋은 캐릭터는 강점 하나, 약점 하나, 보상 성향 하나가 선명해야 한다.
- 캐릭터 설명을 읽고 "어떤 샷을 하고 싶어지는지" 떠올라야 한다.
- 캐릭터 전용 보상은 5-8개면 충분하다.
- 캐릭터가 특정 Shell만 쓰게 만들지 말고, 어떤 Shell을 먹어도 자기 방식으로 바꾸게 만든다.

## 7. Weapon 설계

Weapon은 캐릭터성을 대체하지 않는다. Weapon은 전투에서 쓰는 동사다.

Weapon의 역할:

- Standard: 기준 피해.
- Drill: 지형을 판다.
- Builder: 지형을 만든다.
- Split: 충돌 후 탄을 나눈다.
- Cluster: 여러 표적을 압박한다.
- Precision: 작은 반경과 높은 직격 보상.
- Air: 엄폐 뒤 적을 위에서 친다.
- Roller/Bounce: 지형 형태를 이용한다.
- Hazard: 일정 지역을 위험하게 만든다.
- Control: 적 위치나 탄 경로를 바꾼다.

추천 슬롯 구조:

| 슬롯 | 교체 여부 | 역할 |
| --- | --- | --- |
| Signature Core | 교체 불가 | 캐릭터 정체성. 항상 남는다. |
| Shell Slot 1 | 교체 가능 | 기본 공격 선택지. |
| Shell Slot 2 | 교체 가능 | 보조 문제 해결. |
| Shell Slot 3 | 해금/보상 | 후반 빌드 확장. |
| Item Slot | 소모품 | 한 턴의 위기 대응. |

Shell 보상은 세 종류로 나눈다.

| 보상 | 설명 | 예시 |
| --- | --- | --- |
| New Shell | 새 Shell을 슬롯에 넣는다. | Split Shell 획득. |
| Shell Upgrade | 기존 Shell을 강화/변형한다. | Drill Shell이 더 깊게 판다. |
| Shell Infusion | 현재 캐릭터 Core와 Shell을 섞는다. | Builder 탄이 Split child마다 작은 발판 생성. |

중요:

- New Shell만 계속 주면 캐릭터성이 흐려진다.
- Shell Upgrade와 Shell Infusion이 많아야 런이 자기만의 모양을 가진다.
- Signature Core는 런 중 강화 대상이어야 한다. 그래야 캐릭터를 고른 의미가 유지된다.

## 8. Perk 설계

Perk은 캐릭터와 Weapon 사이의 접착제다.

Perk 타입:

| 타입 | 역할 | 예시 |
| --- | --- | --- |
| Character Perk | 캐릭터 고유 규칙 강화 | Miner: 지형 파괴량에 따라 보너스. |
| Shell Perk | 특정 Shell 계열 강화 | Split child +1, Drill 반경 증가. |
| Condition Perk | 조준/거리/높이 조건 보상 | 낮은 각도 명중 시 추가 피해. |
| Terrain Perk | 지형 파괴/생성 상호작용 | 발판 생성 시 방어막. |
| Economy Perk | 보상/상점/아이템 조작 | 보상 선택지 +1 대신 체력 감소. |
| Risk Perk | 강한 효과와 손해 | 모든 탄 피해 증가, 착탄 시 내 지형도 더 파괴. |
| Rule Perk | 전투 규칙 변경 | 첫 발은 바람 무시, 마지막 발은 두 번 폭발. |

좋은 Perk 예시:

| 이름 | 희귀도 | 효과 |
| --- | --- | --- |
| Low Angle Doctrine | Rare | 25도 이하로 명중하면 피해와 넉백 증가. |
| Excavation Chain | Epic | Drill 효과로 적 아래 지형을 파면 다음 샷이 강화된다. |
| Emergency Scaffold | Rare | 체력이 낮을 때 피격되면 작은 발판을 생성한다. |
| Split Payload | Epic | Split child가 현재 Shell의 일부 효과를 상속한다. |
| Unstable Barrel | Legendary | 모든 Shell이 폭발을 한 번 더 일으키지만 자기 주변 지형도 약해진다. |
| Salvage Contract | Rare | 상자 보상이 좋아지지만 전투 시작 체력이 감소한다. |
| Wind Rider | Epic | 바람 방향으로 쏜 탄은 속도/피해 증가, 반대 방향은 감소. |

나쁜 Perk:

- 피해 +5%만 있는 카드.
- 기존 카드보다 숫자만 큰 상위 카드.
- 선택해도 다음 전투에서 행동이 바뀌지 않는 카드.
- 어떤 캐릭터가 먹어도 똑같은 느낌의 카드만 있는 보상 풀.

## 9. 다른 재미 요소

### 맵 안 보상

상자를 단순 랜덤 드롭이 아니라 지형 퍼즐로 둔다.

예:

- 높은 곳에 떠 있는 상자: Builder, 이동, 정밀 조준으로 획득.
- 적 뒤에 있는 상자: 위험을 감수하고 노릴지 선택.
- 지형 안에 묻힌 광맥: Drill 캐릭터가 유리.
- 폭발물 옆 상자: 광역탄이 위험할 수 있음.

### 지형 오브젝트

| 오브젝트 | 효과 |
| --- | --- |
| Explosive Barrel | 맞으면 큰 폭발. 적도 플레이어도 위험. |
| Reinforced Rock | 일반 폭발에는 약하게 파이고 Drill에 잘 파임. |
| Fragile Ledge | 일정 피해 후 붕괴. 낙사 유도. |
| Wind Vent | 탄 경로를 밀어냄. |
| Repair Plate | 위에 있으면 소량 회복, 파괴 가능. |
| Ore Pocket | 파괴하면 돈/상자/보상 조각 획득. |

### Route 선택

스테이지 사이에 간단한 경로 선택을 둔다.

예:

- 쉬운 전투 + 작은 보상
- 어려운 전투 + 희귀 보상
- 상점
- 이벤트
- 회복
- 엘리트

이 선택은 Slay the Spire식 큰 맵까지 갈 필요는 없다. 데모에서는 다음 두 선택지만 보여줘도 충분하다.

### 이벤트

이벤트는 글만 긴 선택지가 아니라 빌드에 영향을 줘야 한다.

예:

- Weapon Forge: Shell 하나를 강화하거나 다른 계열로 섞는다.
- Black Market: 체력을 지불하고 희귀 Perk 획득.
- Salvage Field: 지형을 많이 파괴할수록 보상 증가.
- Rival Challenge: 선택하면 라이벌과 싸우고 캐릭터 전용 보상 획득.
- Weather Station: 다음 2스테이지의 바람 규칙 선택.

### 스타일 보너스

전투 후 작은 보너스 조건을 둔다.

예:

- Direct Hit: 직격 보너스.
- Terrain Kill: 지형 붕괴/낙사 처치 보너스.
- Multi Kill: 한 발로 2명 이상 처치.
- No Movement: 이동 없이 클리어.
- Risk Shot: 자기 체력 30% 이하에서 처치.

보너스는 큰 성장을 주면 강제 플레이가 되므로, 돈/상자 확률/소량 회복 정도가 적당하다.

### Boss 설계

보스는 체력 많은 적이 아니라 지형 룰을 바꾸는 존재여야 한다.

예:

- Quarry King: 매 턴 지형 일부가 무너진다.
- Fortress Engine: 방벽을 계속 생성한다.
- Wind Tyrant: 풍향이 매 턴 크게 바뀐다.
- Mirror Rival: 플레이어가 많이 쓴 Shell 계열을 흉내 낸다.

## 10. 우선순위

### Demo MVP

데모에서 필요한 최소 재미:

- 플레이어 캐릭터 4명: Miner, Engineer, Gunslinger, Scavenger
- 기본 Shell 6개: Standard, Heavy, Drill, Builder, Split, Cluster
- Perk 24개: 공용 12개, 캐릭터 전용 각 3개
- 적 8종: Grunt, Mortar, Burrower, Builder, Scout, Sniper, Bomber, Shield Carrier
- 맵 템플릿 6개
- 보스 1종
- 이벤트 3종: Forge, Black Market, Weather Station
- 7스테이지 런

### 만들지 말아야 할 것

- 초반부터 40개 이상의 Shell.
- 플레이어 캐릭터와 같은 복잡도의 일반 적.
- 순수 랜덤 맵만으로 구성된 스테이지.
- 모든 캐릭터가 같은 보상 풀만 보는 구조.
- 캐릭터성을 시작 무기 하나에만 의존하는 구조.

## 11. 구현 방향

현재 구조를 바탕으로 추가하면 좋은 데이터 개념:

| 개념 | 필요 이유 |
| --- | --- |
| Character Core Passive | 캐릭터 고유 규칙을 런 내내 유지. |
| Signature Shell/Core Slot | 교체되지 않는 캐릭터 정체성. |
| Character Reward Bias | 캐릭터별 보상 풀 가중치. |
| Character Exclusive Perk Pool | Isaac식 캐릭터별 빌드 감각. |
| Enemy Archetype Definition | 플레이어 캐릭터와 분리된 적 행동. |
| Encounter Definition | 적 조합, 지형 템플릿, 목표를 묶음. |
| Stage Modifier | 바람, 지형 내구도, 특수 오브젝트 같은 방 규칙. |
| Shell Upgrade Chain | Shell을 버리는 대신 변형/강화하는 보상 체인. |

기존 `WeaponDefinition`, `PerkDefinition`, `RewardChoice`, `ShotModifier`, `ProjectileEffects`는 유지해도 된다. 다만 캐릭터성을 살리려면 캐릭터 쪽에 `고유 규칙`, `고정 Core`, `보상 성향`이 추가되어야 한다.

## 12. 최종 기준

이 방향이 맞는지 판단하는 질문:

- 같은 Shell을 먹었을 때 캐릭터마다 다른 판단이 생기는가?
- 스테이지마다 "이번에는 무엇을 먼저 해야 하지?"라는 질문이 생기는가?
- 적 조합 때문에 무기 선택과 이동 선택이 바뀌는가?
- Perk 선택 후 다음 샷 방식이 달라지는가?
- 캐릭터를 다시 골랐을 때 같은 게임이 아니라 다른 런처럼 느껴지는가?

이 질문에 "아니오"가 많으면 콘텐츠 수를 늘려도 재미는 늘지 않는다.
